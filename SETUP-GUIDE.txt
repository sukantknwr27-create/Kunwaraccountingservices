STEP 1 — RUN DATABASE SQL
─────────────────────────
1. Go to supabase.com → Your project
2. Click SQL Editor → New query
3. Open database-v3.sql from this ZIP
4. Copy ALL content → Paste → Click Run
5. You should see 6 tables created with 0 rows each ✅

STEP 2 — CREATE SUPABASE STORAGE BUCKET
────────────────────────────────────────
1. Go to Supabase → Storage (left sidebar)
2. Click "New bucket"
3. Bucket name: client-documents
4. Toggle: Private (NOT public) ✅
5. Click Create bucket
6. Click on the bucket → Policies → New policy
7. Select "For full customization"
8. Policy name: allow_all_storage
9. Allowed operations: SELECT, INSERT, UPDATE, DELETE
10. USING expression: true
11. WITH CHECK expression: true
12. Click Save ✅

STEP 3 — ENABLE FIREBASE AUTHENTICATION
────────────────────────────────────────
1. Go to console.firebase.google.com
2. Open project "kunwar-accounting"
3. Left sidebar → Authentication → Sign-in method
4. Enable GOOGLE:
   - Click Google → Enable toggle ON
   - Project support email: sukant@kunwaraccountingservices.in
   - Click Save ✅
5. Enable PHONE:
   - Click Phone → Enable toggle ON
   - Click Save ✅
6. Enable EMAIL/PASSWORD:
   - Click Email/Password → Enable toggle ON
   - Click Save ✅

STEP 4 — ADD YOUR DOMAIN TO FIREBASE
──────────────────────────────────────
1. Firebase → Authentication → Settings → Authorized domains
2. Click "Add domain"
3. Add: kunwaraccountingservices.in
4. Add: www.kunwaraccountingservices.in
5. Click Add ✅
(Without this, Google Sign-In will be blocked)

STEP 5 — UPDATE ANON KEY IN ALL FILES
───────────────────────────────────────
Your Supabase anon key needs to be in 3 files:

1. Get your anon key:
   Supabase → Project Settings → API → "anon public" key (starts with eyJ...)

2. Open supabase.js → Replace:
   PASTE_YOUR_ANON_KEY_HERE → your actual key

3. Open firebase.js → Replace:
   PASTE_YOUR_ANON_KEY_HERE → your actual key (line ~15)

4. Open dashboard.html → Replace ALL 3 occurrences of:
   PASTE_YOUR_ANON_KEY_HERE → your actual key
   (Search for it in the file — Ctrl+F)

================================================================
COMMON ISSUES & FIXES
================================================================

Issue: Google Sign-In popup blocked
Fix: Add your domain to Firebase Authorized Domains (Step 4)

Issue: OTP not received
Fix: Firebase Phone auth needs billing enabled for production
     Go to Firebase → Upgrade to Blaze plan (free tier included)
     Or test with Firebase test phone numbers first

Issue: Document upload fails  
Fix: Check Supabase Storage bucket policies (Step 2)
     Make sure bucket name is exactly: client-documents

Issue: Login works but dashboard shows empty
Fix: Check if database-v3.sql was run successfully
     Go to Supabase → Table Editor → users table
     Should have a row after first login

Issue: Tool gate not showing
Fix: Clear browser localStorage and try again
     Open DevTools → Application → Local Storage → Clear All

HOW NOTIFICATIONS WORK
================================================================

Automatic notifications are created when:
1. A filing due date is 7 days away
2. A filing due date is 2 days away  
3. A filing becomes overdue

The system checks deadlines automatically when client logs in.
Notifications appear in:
  → Dashboard → Notifications tab (with unread count badge)
  → Browser push notification (if client allows)
  → Overview tab (recent 3 notifications)

For WhatsApp reminders — send manually from admin panel
using the client's phone number from users table.
